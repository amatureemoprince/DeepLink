# 详细需求

在实际场景中，需要根据用户的角色和级别对数据访问权限的控制，这也是一个难点，此项目采用`RBAC`实现对不同角色的权限区分，`标签`实现数据访问
权限隔离。将这两种方式结合可以实现“横向”和“纵向”上的权限隔离。

# 接口设计

## 1. 用户登录 [Login]

---

### 接口说明

用于用户身份验证并获取 JWT 令牌。

### 登录接口

- **Method**: `POST`
- **URL**: `/api/auth/login`

### 请求参数

| 字段       | 类型     | 必填 | 说明  |
|:---------|:-------|:---|:----|
| username | string | 是  | 用户名 |
| password | string | 是  | 密码  |

### 返回结果
```json
{
    "code": 200,
    "data": { "token": "..." },
    "msg": "操作成功"
}
```
